# 05_구현백로그

- 작성일: 2026-03-10
- 상태: 활성 SSOT

## 1. 개발 원칙

- 한 번에 한 유스케이스만 구현한다.
- 한 번에 한 도메인만 크게 수정한다.
- 구현 전에 관련 `/docs/01~05`를 먼저 갱신한다.
- 구현 후 바로 실행, 테스트, 수동 검증을 한다.
- MVP1 범위를 벗어나면 바로 backlog로 보낸다.

## 2. 우선순위

### P0. 반드시 끝까지 연결할 핵심 유스케이스

- 카카오 로그인
- 홈 featured 코스 조회
- 홈 주변 화장실 조회
- 코스 상세 조회
- 코스 따라가기 L2
- 라이딩 기록 저장
- 라이딩 기반 코스 생성
- 코스 공유 링크 발급/조회

### P1. 현재 코드/설계는 있으나 MVP1 완료 기준에서는 분리할 항목

- 컬렉션 생성/코스 추가
- 코스 하이라이트 조회/생성
- 코스모임 생성/참가/탈퇴
- 모임 채팅
- GPX 직접 업로드

### P2. 이후 단계

- 개인화 추천
- LLM 추천 에이전트
- 턴바이턴
- 오프라인 패키지
- 운영 대시보드

## 3. 작업 단위 예시

### TASK-00 문서/계측/운영정책 잠금

- 목적: 구현 전에 PRD, API 계약, 정책 문서의 성공 기준과 예외 규칙을 먼저 고정
- 완료 기준:
  - `docs/01`에 운영 게이트가 숫자로 명시됨
  - `docs/04`에 MVP1 이벤트 이름이 고정됨
  - `설계/04`의 인증/공유/GPS 정책이 현재 구현과 충돌하지 않음

### TASK-000 쓰기 API 인증/소유권 정리

- 목적: 라이딩 저장/코스 생성/공유 발급에서 클라이언트 입력 대신 JWT로 소유권을 확정
- 완료 기준:
  - `POST /api/v1/ridings` 인증 필요
  - `POST /api/v1/courses/from-riding` 인증 필요
  - 요청 바디 `userId`, `ownerUserId` 제거 또는 무시
  - 타인 소유 riding/course 접근 시 `403`

### TASK-001 카카오 로그인 완료 플로우

- 목적: 카카오 authorization code -> 서비스 JWT 발급 -> 인증 상태 확정
- 입력: `code`, `codeVerifier`, `redirectUri`, `nonce`
- 완료 기준:
  - 로그인 성공
  - 프론트의 `/api/auth/me` 재검증 성공
  - 토큰 만료/오류 처리

### TASK-002 홈 featured 코스 조회

- 목적: 홈 진입 시 featured 코스 목록 표시
- 완료 기준:
  - 목록 조회 성공
  - 빈 목록 처리
  - fallback 사용 시 사용자 표시 유지

### TASK-003 홈 주변 화장실 조회

- 목적: 현재 위치 기준 화장실 조회
- 완료 기준:
  - 위치 권한 허용 시 조회 성공
  - 권한 거부 처리
  - 조회 실패 시 코스 탐색은 계속 가능

### TASK-004 코스 상세 조회

- 목적: 코스 메타데이터, path, warnings, amenitiesSummary 노출
- 완료 기준:
  - 상세 조회 성공
  - not found 처리
  - 태그/경고/path 렌더링 확인

### TASK-005 코스 따라가기 L2

- 목적: 코스 path 기반 진행률/이탈 안내 화면 제공
- 완료 기준:
  - 상세에서 가이드 화면으로 이동
  - 진행률 표시
  - 오프루트 상태 표시
  - 위치 미확보 시 graceful handling

### TASK-006 라이딩 기록 저장

- 목적: 주행 path를 저장 가능한 형태로 수집/전송
- 완료 기준:
  - 기록 시작/종료
  - 최소 좌표 검증
  - 로그인 상태에서만 저장
  - 저장 성공/실패 처리

### TASK-007 라이딩 기반 코스 생성

- 목적: 저장된 ridingId로 코스 생성
- 완료 기준:
  - 생성 성공
  - riding 없음 처리
  - 타인 riding 차단
  - visibility/title 검증

### TASK-008 공유 링크 발급과 공개 코스 조회

- 목적: 로그인 사용자가 shareId를 만들고, 다른 사용자가 공개/링크공유 코스를 연다
- 완료 기준:
  - share 발급 성공
  - public/unlisted 조회 성공
  - `/share/[shareId]` 진입 경로 정의 또는 구현
  - private 차단 정책 확인
