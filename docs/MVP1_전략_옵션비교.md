# MVP1 배포 전략 옵션 비교 (Frontend + Backend + CI/CD)

작성일: 2026-03-05  
목적: 전략 선택 시 장단점을 문서로 남겨 의사결정을 재사용한다.

---

## 1) 프론트 배포 전략

### A. Expo Web on EC2
- 장점
  - 백엔드와 같은 도메인/환경에서 즉시 E2E 가능
  - 운영비용 낮고 단순(정적 파일 + Nginx)
  - GitHub Actions와 결합이 쉬움
- 단점
  - 모바일 네이티브 UX/성능을 완전히 대체하진 못함

### B. 모바일 앱 빌드만
- 장점
  - 실사용과 가장 가까운 형태
- 단점
  - 배포/리뷰 속도가 느리고 QA 진입 장벽이 큼
  - 백엔드 연동 이슈를 빠르게 반복 검증하기 어려움

### C. 둘 다
- 장점
  - 웹에서 빠른 반복 + 모바일 실제성 확보
  - MVP1 검증 폭이 가장 넓음
- 단점
  - 빌드/배포 파이프라인 관리 포인트 증가

---

## 2) CI/CD 전략

### A. GitHub Actions + SSH Deploy
- 장점
  - 현재 GitHub 기반 저장소와 바로 연결
  - 서버 접근만 되면 빠르게 자동화 가능
  - 초기 구축 난이도/비용이 낮음
- 단점
  - 서버 상태에 따라 배포 성공률이 좌우됨

### B. GitHub Actions + Registry (ECR/DockerHub)
- 장점
  - 이미지 기반으로 재현성 높음
  - 다중 서버 확장에 유리
- 단점
  - 레지스트리 권한/보안/비용 관리 필요

### C. Jenkins on EC2
- 장점
  - 고급 파이프라인 커스터마이징 가능
- 단점
  - 운영 오버헤드가 큼(백업/업데이트/보안패치)

---

## 3) 배포 전략

### A. Docker Compose + Blue/Green Lite
- 장점
  - 롤백이 빠름(트래픽 포인터만 전환)
  - 다운타임 최소화 가능
- 단점
  - 구성 파일/스크립트 복잡도 증가

### B. 단일 컨테이너 재시작
- 장점
  - 가장 단순
- 단점
  - 배포 시점 다운타임 가능성

### C. systemd 직접 실행
- 장점
  - Docker 미사용 환경에서 단순
- 단점
  - 의존성/환경 재현성 저하

---

## 4) DB 전략

### A. EC2 내 Postgres
- 장점
  - 빠르고 저렴
- 단점
  - 백업/복구/장애대응 부담 큼

### B. RDS(PostgreSQL + PostGIS)
- 장점
  - 운영 안정성/백업/모니터링 강점
  - 앱 서버와 역할 분리 용이
- 단점
  - 비용 증가, 초기 설정 복잡

---

## 5) HTTPS 전략

### A. Nginx + Let's Encrypt
- 장점
  - 실제 운영과 동일한 표준 경로
  - OAuth redirect, 보안 헤더 적용에 유리
- 단점
  - 도메인/DNS/인증서 갱신 운영 필요

### B. HTTP만
- 장점
  - 빠른 초기 검증
- 단점
  - OAuth/실서비스 검증 제약 큼

---

## 6) GPX 저장 전략

### A. DB 본문 저장(`gpx_data`)
- 장점
  - 구현 단순
  - 트랜잭션 경계가 명확
- 단점
  - DB 용량/백업 비용 증가
  - 대용량 GPX 증가 시 조회/복구 부담 증가

### B. S3 본문 저장 + DB 키 저장
- 장점
  - DB는 메타데이터 중심으로 가벼워짐
  - 저장소 비용/확장성 측면 유리
  - CDN/수명주기 정책으로 운영 유연성 확보
- 단점
  - S3 권한/IAM 설정이 필요
  - 스토리지 장애 시 조회 실패 가능성 고려 필요

---

## 7) 모니터링 도입 전략

### A. MVP1부터 전체 도입(Prometheus + Loki + Grafana)
- 장점
  - 초기에 관측 공백이 거의 없음
- 단점
  - 초기 구축/운영 부담이 커짐

### B. 2단계 도입(배포 안정화 후 추가)
- 장점
  - MVP1 배포 속도 유지
  - 우선순위(기능 검증 > 관측 확장) 관리가 쉬움
- 단점
  - 초반에는 최소 로그/헬스체크에 의존해야 함

---

## 최종 채택 (MVP1)
- 프론트: **둘 다(Expo Web + 모바일 빌드)**
- CI/CD: **GitHub Actions + SSH Deploy**
- 배포: **Docker Compose + Blue/Green Lite**
- DB: **RDS(PostgreSQL/PostGIS)**
- HTTPS: **Nginx + Let's Encrypt**
- GPX 저장: **S3 본문 + DB 키 저장**
- 모니터링: **2단계 도입(배포 안정화 후 Prometheus/Loki/Grafana)**
