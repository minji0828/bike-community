# 01_MVP1_제품요구사항

- 작성일: 2026-03-10
- 상태: 활성 SSOT

## 1. 제품 한 줄 설명

- 이 서비스는 라이딩 코스를 찾거나 직접 기록해 저장하고 싶은 사용자가, 코스와 경로 주변 편의시설을 확인하며 쉽게 따라가고 공유하도록 돕는 서비스다.

## 2. 해결하려는 문제

- 사용자는 탈 만한 코스를 바로 찾기 어렵다.
- 라이딩 중 필요한 편의시설, 특히 화장실을 현재 경로 기준으로 확인하기 어렵다.
- 내가 탄 길을 다시 열어보거나 다른 사람에게 공유하기 어렵다.
- 기존 대안은 운동 기록 또는 순수 지도 탐색에 치우쳐 있어, "코스 콘텐츠 + 경로 주변 편의 + 따라가기"가 한 번에 이어지지 않는다.

## 3. 타겟 사용자

- 핵심 사용자:
  - 주말에 가볍게 라이딩하거나 산책형 코스를 찾는 사용자
  - 내가 탄 길을 저장하고 다시 열어보려는 사용자
- 보조 사용자:
  - 여행형 대표 코스를 열어보는 사용자
  - 코스를 링크로 공유받아 확인하는 사용자
- 제외 사용자:
  - 턴바이턴 내비게이션을 기대하는 하드코어 내비 사용자
  - 커뮤니티/챌린지/소셜 피드 중심 서비스를 기대하는 사용자

## 4. MVP1 목표

- MVP1에서 검증할 핵심 가설:
  - 사용자는 "코스를 열고 따라가며, 필요할 때 경로 주변 화장실을 확인하는 경험"만으로도 서비스 가치를 느낀다.
  - 사용자는 "라이딩 기록 -> 코스 저장 -> 공유" 흐름을 실제로 반복한다.
- 사용자가 반드시 하게 만들 핵심 행동 1개:
  - 코스 상세를 열고 `따라가기`를 시작한다.
- 성공 기준:
  - `course_detail_viewed` 이벤트가 100건 이상 수집된다.
  - `guide_started / course_detail_viewed` 비율이 20% 이상이다.
  - `riding_save_succeeded / riding_save_requested` 비율이 90% 이상이다.
  - `shared_course_opened` 이벤트가 30건 이상 수집된다.

메모:

- 위 수치는 MVP1의 1차 운영 게이트다.
- 실제 운영 데이터 100건 이상이 쌓이면 다음 릴리즈에서 재조정할 수 있다.

## 5. MVP1 포함 범위

- 카카오 로그인과 로그인 상태 확인
- 홈에서 featured 코스 조회
- 홈에서 내 주변 화장실 조회
- 코스 상세 조회
- L2 코스 따라가기 화면
- 라이딩 기록 저장
- 라이딩 기반 코스 생성
- 코스 공유 링크 발급
- 공개/링크공유 코스 조회

## 6. MVP1 제외 범위

- 턴바이턴 내비게이션
- 재탐색, 음성 안내, 오프라인 패키지
- 개인화 추천, LLM 추천 에이전트
- 커뮤니티 풀세트, 댓글/신고/모더레이션
- 컬렉션, 하이라이트, 코스모임, 실시간 채팅
- 관리자 UI, 고급 통계
- 외부 결제

메모:

- 컬렉션, 하이라이트, 코스모임, 채팅은 코드와 설계가 일부 존재하지만 MVP1 핵심 경로의 완료 기준에는 넣지 않는다.

## 7. 핵심 사용자 시나리오

### 시나리오 1. 기본 코스를 열고 따라가기

- 사용자는 홈에 진입한다.
- 사용자는 featured 코스를 본다.
- 사용자는 코스 상세를 연다.
- 사용자는 따라가기를 시작한다.
- 사용자는 진행률과 이탈 안내를 확인한다.

### 시나리오 2. 라이딩 기록을 코스로 저장하기

- 사용자는 로그인한다.
- 사용자는 라이딩을 시작한다.
- 사용자는 라이딩을 종료하고 저장한다.
- 사용자는 저장된 라이딩으로 코스를 생성한다.
- 사용자는 생성된 코스 상세를 다시 연다.

### 시나리오 3. 코스 공유 링크 열기

- 사용자는 코스 공유 링크를 발급한다.
- 다른 사용자는 링크로 공개/링크공유 코스를 연다.
- 다른 사용자는 코스 상세를 확인한다.

## 8. 완료 기준

- 시나리오 1, 2, 3이 실제 API와 프론트에서 동작한다.
- 현재 MVP1에 필요한 주요 API가 `/docs/04_API_상태계약.md`와 일치한다.
- 주요 예외 흐름이 UI와 API 양쪽에서 처리된다.
- 보류 기준:
  - `course_detail_viewed` 100건 이후에도 `guide_started / course_detail_viewed`가 10% 미만이면 가치 제안을 재검토한다.
  - `riding_save_succeeded / riding_save_requested`가 80% 미만이면 기능 확장보다 저장 안정화가 우선이다.
- 최소 검증:
  - 백엔드 관련 테스트
  - 프론트 `lint` / `typecheck`
  - 핵심 흐름 수동 점검

## 9. 오픈 전 확인사항

- 카카오 로그인 redirect / 환경변수 점검
- 개인정보 및 위치 정보 고지 문구
- 공개/비공개/링크공유 코스 노출 정책
- 서버 에러 로그 확인 가능 여부
- 운영 최소 권한과 관리자 키 관리 방식
