# 외부 테스터 공개 테스트 최소 행동 체크리스트

- 작성일: 2026-03-11
- 대상: 저장소 오너

이 문서는 "내가 직접 해야 하는 최소 행동"만 적는다.

## 1. Render

- Render에서 `New Blueprint`를 누른다.
- 이 저장소를 연결한다.
- `render.yaml`을 그대로 사용한다.
- 생성 후 DB SQL Shell에서 아래 1줄만 실행한다.

```sql
CREATE EXTENSION IF NOT EXISTS postgis;
```

## 2. Render 환경변수 입력

- `SEOUL_API_KEY`
- `KAKAO_REST_API_KEY`
- `KAKAO_CLIENT_SECRET`
- `KAKAO_ALLOWED_REDIRECT_URIS`
- `APP_CORS_ALLOWED_ORIGINS`

메모:

- `APP_JWT_SECRET`, `ADMIN_API_KEY`는 자동 생성되게 준비해두었다.
- DB 연결 URL은 `render.yaml` + entrypoint에서 자동 조립되게 준비해두었다.

## 3. Vercel

- Vercel에서 `frontend/`를 root directory로 import 한다.
- 아래 환경변수만 넣는다.

- `NEXT_PUBLIC_API_BASE_URL`
- `NEXT_PUBLIC_KAKAO_CLIENT_ID`
- `NEXT_PUBLIC_KAKAO_MAP_JS_KEY`

필요하면 추가:

- `NEXT_PUBLIC_KAKAO_REDIRECT_URI`

## 4. Kakao Developers

- 플랫폼/사이트 도메인에 Vercel URL을 등록한다.
- Redirect URI에 아래를 등록한다.

```text
https://<your-vercel-domain>/auth/kakao/callback
```

## 5. 끝나면 나에게 줄 것

- Vercel 배포 URL
- Render API URL

이 두 개만 알면 나머지 점검은 내가 계속 이어서 할 수 있다.
