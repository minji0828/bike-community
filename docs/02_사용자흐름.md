# 02_사용자흐름

- 작성일: 2026-03-10
- 상태: 활성 SSOT

## 1. 전체 사용자 흐름

### 비로그인 사용자

- 홈 진입
- featured 코스 조회
- 코스 상세 조회
- 공유 링크 코스 조회
- 로그인 필요 기능에서 카카오 로그인 유도

### 로그인 사용자

- 홈 진입
- 내 주변 화장실 조회
- 코스 상세 조회
- 따라가기 시작
- 라이딩 기록 저장
- 라이딩 기반 코스 생성
- 코스 공유 링크 발급
- 공유 링크로 들어온 코스를 다시 따라가기

### 운영자

- 화장실 데이터 동기화
- 피처드 코스 관리

## 2. 화면 목록

- 홈 화면 `/`
- 프로필/로그인 화면 `/profile`
- 코스 상세 화면 `/course/[id]`
- 코스 따라가기 화면 `/course/[id]/guide`
- 라이딩 기록 화면 `/ride`
- 공유 코스 진입 화면 `/share/[shareId]`

## 3. 핵심 플로우 상세

### 플로우 A. 홈에서 코스를 열고 따라가기

1. 사용자가 홈에 진입한다.
2. featured 코스 목록을 조회한다.
3. 코스를 선택해 상세를 연다.
4. 코스 경로, 태그, 경고, 편의 요약을 확인한다.
5. `따라가기`를 눌러 가이드 화면으로 이동한다.
6. 진행률, 경과 시간, 이탈 여부를 확인한다.

### 플로우 B. 로그인 후 라이딩 저장 -> 코스 생성

1. 사용자가 카카오 로그인한다.
2. 라이딩 화면으로 이동한다.
3. 위치 권한을 허용하고 기록을 시작한다.
4. 라이딩을 종료하고 저장한다.
5. 저장된 라이딩으로 코스를 생성한다.
6. 생성된 코스 상세를 확인한다.

### 플로우 C. 코스 공유

1. 로그인 사용자가 코스 상세에서 공유 링크를 발급한다.
2. 링크를 전달한다.
3. 상대방이 공유 링크로 코스를 연다.
4. 공개 범위 정책에 맞으면 코스 상세를 확인한다.

## 4. 예외 플로우

- 비로그인 사용자가 보호 API를 호출하면 로그인 유도
- 비로그인 사용자가 라이딩 저장 또는 코스 생성을 시도하면 로그인 유도 후 현재 흐름을 중단
- 위치 권한 거부 시 홈 주변 화장실/라이딩 기록 기능 제한
- 코스가 없으면 상세 화면에서 not found 처리
- path 좌표가 2개 미만이면 라이딩 저장 또는 코스 생성 실패
- 토큰 만료 시 인증 상태를 해제하고 재로그인 유도
- private 코스는 공유 링크로 직접 노출하지 않음
- 공유 링크 페이지는 읽기 전용 진입이며, 쓰기 기능은 로그인 상태와 소유권이 확인될 때만 노출
- 경로 주변 조회 요청에 path가 비면 빈 결과 또는 실패 정책을 명확히 적용

## 5. 화면별 상태 정의

### 홈 화면

- 초기
- 코스 로딩
- 코스 정상 조회
- 코스 fallback 표시
- 위치 권한 거부
- 주변 화장실 조회 실패

### 코스 상세 화면

- 로딩
- 정상 조회
- 코스 없음
- highlights / meetups 섹션 부분 실패
- 따라가기 가능

### 따라가기 화면

- 로딩
- 가이드 준비
- 진행 중
- 일시정지
- 경로 이탈
- 종료

### 라이딩 화면

- 초기
- 위치 권한 대기
- 기록 중
- 일시정지
- 저장 중
- 저장 완료
- 저장 실패

### 인증 상태

- checking
- authenticated
- anonymous
- error
