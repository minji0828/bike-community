# 외부 테스터 집에서 테스트 전체 계획

- 작성일: 2026-03-11
- 상태: 실행 계획

목적:

- 실제 의뢰자 또는 외부 테스터가 집에서 URL만 열어 서비스 핵심 흐름을 테스트할 수 있게 한다.
- 사용자가 직접 해야 할 행동은 계정 연결, 비밀값 입력, 카카오 설정 같은 최소 단계만 남긴다.

---

## 1. 목표 상태

- 웹 URL 하나로 홈, 코스 상세, 공유 코스 열람, 로그인, 라이딩 저장 흐름을 테스트할 수 있다.
- 백엔드는 외부 인터넷에서 접근 가능한 고정 URL을 가진다.
- 카카오 로그인 redirectUri가 실제 배포 URL과 일치한다.
- 의뢰자는 복잡한 로컬 세팅 없이 브라우저만으로 1차 검증이 가능하다.

---

## 2. 배포 구조

- 웹: Vercel
- 백엔드: Render Web Service
- DB: Render Postgres + PostGIS

도메인 1차안:

- 웹: `https://<project>.vercel.app`
- API: `https://bikeoasis-api.onrender.com`

메모:

- 커스텀 도메인은 나중 단계다.
- 1차 목표는 집에서 바로 열리는 공개 URL을 확보하는 것이다.

---

## 3. 내가 repo에 준비해둘 것

- `render.yaml` 블루프린트
- Vercel/Render 환경변수 문서
- backend CORS 설정
- Kakao redirectUri/allowed origin 문서
- 홈테스트 체크리스트
- 핵심 smoke test 순서

현재 완료 상태:

- Render 블루프린트 파일 추가 완료
- CORS exact origin + origin pattern 지원 추가 완료
- 홈테스트 전략/체크리스트 문서 추가 완료

---

## 4. 실제 실행 순서

### Step 1. GitHub 기준 배포 준비

- 저장소 최신 상태를 push
- Render는 `render.yaml` 기준으로 서비스/DB를 만든다
- Vercel은 `frontend/` 루트를 기준으로 프로젝트를 만든다

### Step 2. 백엔드 먼저 배포

- Render에서 DB + backend 서비스 생성
- PostGIS extension 활성화
- backend health URL 확인

### Step 3. 웹 배포

- Vercel에서 `frontend/`를 배포
- `NEXT_PUBLIC_API_BASE_URL`을 Render URL로 지정
- 웹 URL 확보

### Step 4. 카카오 설정

- 웹 URL 기준 redirectUri 등록
- 필요 시 모바일 딥링크는 후속 등록

### Step 5. 홈테스트 점검

- 홈 진입
- featured 코스 열람
- 공유 코스 열람
- 카카오 로그인
- 라이딩 저장

---

## 5. 사용자 최소 행동

사용자가 직접 해야 하는 행동은 아래만 남긴다.

1. Render와 Vercel에 GitHub 저장소를 연결한다.
2. 비밀값을 입력한다.
   - Kakao REST API Key
   - Kakao Client Secret
   - Seoul API Key
3. 카카오 개발자 콘솔에 실제 배포 URL redirectUri를 등록한다.
4. Render DB에서 `CREATE EXTENSION IF NOT EXISTS postgis;`를 1회 실행한다.

그 외:

- 배포 구조
- repo 설정
- 환경변수 표
- CORS/redirectUri 기준
- 문서 정리
는 내가 이미 진행하거나 repo 안에 준비한다.

---

## 6. 의뢰자 테스트 시나리오

### 시나리오 A. 읽기 테스트

- 홈 접속
- featured 코스 클릭
- 코스 상세 확인
- 공유 링크 열람

### 시나리오 B. 로그인 테스트

- 카카오 로그인
- 프로필/세션 상태 확인

### 시나리오 C. 쓰기 테스트

- 라이딩 저장
- 저장한 라이딩을 코스로 전환
- 공유 링크 발급

---

## 7. 성공 기준

- 외부 테스터가 로컬 세팅 없이 공개 URL만으로 핵심 흐름을 테스트한다.
- 카카오 로그인과 코스 읽기/쓰기 흐름이 배포 환경에서 동작한다.
- 문제 발생 시 어떤 플랫폼(Vercel/Render/Kakao/DB)에서 터졌는지 바로 구분할 수 있다.
