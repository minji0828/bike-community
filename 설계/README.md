# 설계 문서 모음

- 마지막 검토: 2026-03-10
- 목적: 어떤 문서가 실제 구현 계약(SSOT)인지 빠르게 구분하게 한다.

이 폴더는 기획, 도메인 탐색, MVP 계약, 확장 전략이 함께 들어 있다.
중복이 있기 때문에 모든 문서를 같은 비중으로 읽으면 안 된다.

## 현재 저장소 기준 현실

- 백엔드: Spring Boot + Postgres/PostGIS
- 프론트엔드: `frontend/`의 Next.js 웹 앱
- 문서 안의 React Native / Expo / RN 표기는 "제품 목표/모바일 기준 참고"가 섞여 있다.
- 현재 저장소를 바로 개발할 때는 웹 계약을 우선으로 보고, 모바일 전용 내용은 별도 확정 없이 그대로 구현 기준으로 쓰지 않는다.

## 문서 작성 원칙

- 문서 상단에 `버전 / 작성일 / 상태`를 남긴다.
- 기능 요구사항은 `FR-###`, 비기능은 `NFR-###`, 정책은 `POL-###`처럼 추적 가능한 ID를 사용한다.
- 새 기능은 먼저 관련 설계 문서를 고친 뒤 구현한다.
- 새 기술이나 외부 연동은 `설계/15_기술스택_레지스트리.md`에 추가한다.
