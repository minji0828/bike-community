# 설계 문서 모음

이 폴더는 1인 개발을 위한 설계 산출물을 모아두는 곳입니다.

현재 의사결정(요약):

- 길안내 MVP: 코스 따라가기 가이드(레벨 2)

- 콜드스타트: 기본 제공 코스(큐레이션) + 코스 메타데이터 v1

작성 원칙:

- 문서 상단에 `버전/작성일/작성자(또는 상태)`를 남깁니다.
- 기능 요구사항은 `FR-###`, 비기능은 `NFR-###`, 정책은 `POL-###`처럼 ID를 부여합니다.
- 결정사항(Decision)은 근거(Rationale)와 함께 남깁니다.
- 구현에 들어가면, 문서와 코드가 어긋나지 않도록 변경 이력을 기록합니다.
- 새 기술(라이브러리/인프라/외부 API/파일 포맷)을 도입하면 `설계/15_기술스택_레지스트리.md`에 사용 근거/리스크를 반드시 추가합니다.

문서 추가 시:

- 가능하면 2자리 번호로 순서가 보이게 파일명을 추가합니다.
- 새 기능을 만들기 전에 반드시 관련 문서를 먼저 갱신합니다(문서가 단일 진실원천).
