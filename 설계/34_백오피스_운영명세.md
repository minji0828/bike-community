## 1. 운영 역할 정의

| 역할 | 설명 | 핵심 권한 |
|---|---|---|
| `ADMIN` | 시스템 설정과 운영 예외 처리 담당 | 전체 조회, 수동 정정, 동기화 실행 |
| `CURATOR` | 피처드 코스와 태그/경고 관리 | 코스 품질 관리, featured 순위 조정 |
| `OPERATOR` | 모니터링과 고객 문의 대응 | 조회 중심, 제한된 상태 조치 |
| `AUDITOR` | 감사 로그 점검 | 읽기 전용 감사 로그 조회 |

## 3. 운영 데이터 수집 항목

### 3-1. 제품 지표

| 데이터 | 설명 | 수집 위치 |
|---|---|---|
| `course_detail_viewed` | 코스 상세 진입 수 | frontend |
| `guide_started` | 따라가기 시작 수 | frontend |
| `riding_save_requested` | 라이딩 저장 시도 수 | frontend |
| `riding_save_succeeded` | 라이딩 저장 성공 수 | backend |
| `course_created_from_riding` | 라이딩 기반 코스 생성 성공 수 | backend |
| `course_share_issued` | 공유 링크 발급 수 | backend |
| `shared_course_opened` | 공유 코스 열람 수 | frontend |

### 3-2. 운영 로그

| 데이터 | 설명 | 보존 기준 |
|---|---|---|
| 인증 실패 로그 | redirectUri, nonce, 토큰 검증 실패 | 보안 로그 |
| 공유 링크 발급 로그 | courseId, ownerUserId, 발생 시각 | 운영 로그 |
| POI 동기화 실행 결과 | mode, 처리 건수, 실패 원인 | 운영 로그 |
| 관리자 조치 이력 | actor, action, target, reason | 감사 로그 |

주의:

- 위치 원본 좌표는 일반 운영 로그에 남기지 않는다.
- 공유 링크 원문 `shareId`는 외부 분석 도구로 직접 보내지 않는다.

---

## 4. 관리자 조치와 감사 로그 정책

- 다음 조치는 감사 로그 대상이다.
  - 코스 비노출/삭제
  - featured 순위 변경
  - 태그 사전 수정
  - 경고 추가/삭제
  - 공유 링크 무효화
  - POI 강제 동기화 실행
- 감사 로그 최소 필드:
  - `actorRole`
  - `actorUserId`
  - `actionType`
  - `targetType`
  - `targetId`
  - `reason`
  - `createdAt`

## 5. 운영 API/화면 권장 구조

| 경로 | 용도 |
|---|---|
| `/admin/dashboard` | 운영 KPI 대시보드 |
| `/admin/courses` | 코스 검색/상태 변경 |
| `/admin/courses/{courseId}` | 상세 운영 |
| `/admin/courses/{courseId}/warnings` | 경고 관리 |
| `/admin/courses/{courseId}/metadata/recalculate` | 메타데이터 재계산 |
| `/admin/tags` | 태그 사전 관리 |
| `/admin/poi/sync` | 동기화 실행/결과 |
| `/admin/ridings` | 라이딩 조회 |
| `/admin/shares` | 공유 링크 현황 |
| `/admin/audit-logs` | 감사 로그 |

## 6. 운영 KPI 기준

| 지표 | 기준 | 운영 액션 |
|---|---|---|
| `guide_started / course_detail_viewed` | 20% 미만 | 코스 상세 CTA/설명 점검 |
| `riding_save_succeeded / riding_save_requested` | 90% 미만 | GPS/저장 오류 우선 점검 |
| POI 동기화 실패 연속 횟수 | 3회 이상 | 운영 경보 및 수동 동기화 검토 |
| 공유 링크 404 비율 | 급증 시 | 공개범위/비노출 정책 점검 |
