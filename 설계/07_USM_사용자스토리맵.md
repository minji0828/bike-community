- 문서 ID: USM
- 버전: v0.1

## 1. 페르소나(초안)

- P1 주말 산책 라이더: 목적지는 없거나 대략만, "불안(화장실)"을 줄이며 편하게 타고 싶다.
- P2 코스 수집가/공유자: 마음에 든 길을 저장하고 재사용/공유하고 싶다.
- P2-2 여행/탐방형 라이더: 검증된 대표 코스를 따라가고, 편의정보와 동행 경험까지 원한다.
- P3 큐레이터(운영): 기본 코스를 등록/태깅/검증해 콜드스타트를 해결한다.

## 2. Backbone(상위 활동)

0) 앱 시작/홈(대시보드)

1) 코스 탐색/선택

2) 코스 상세 확인

3) 라이딩(코스 따라가기)

4) 중간 정차/편의(POI)

5) 기록 저장/코스 생성(UGC)

6) 공유/재사용

7) 여행 계획/컬렉션

8) 내 활동(내 코스/내 라이딩/완주)

9) 운영(피처드/태그/경고/하이라이트)

## 3. 스토리맵(릴리즈 슬라이스)

표기:

- Must(MVP) = 개발 시작 시점에서 "바로 만들어도 된다" 수준의 정의/책임이 있는 항목
- Next = MVP 이후

### 3.0 홈(대시보드)

Must(MVP)

- USM-HOME-001: 홈에서 피처드 코스 카드/리스트로 진입
- USM-HOME-002: 홈에서 "바로 시작" CTA(추천 코스 상세로 이동)
- USM-HOME-003: 홈에서 "라이딩 시작" CTA(기록 화면으로 이동)
- USM-HOME-004: 홈에서 산책형/여행형 Featured 코스를 함께 탐색

Next

- USM-HOME-101: 최근 본 코스/최근 따라가기 이어하기(로컬 상태)

### 3.1 코스 탐색/선택

Must(MVP)

- USM-DISC-001: 기본 제공 코스(피처드) 리스트 보기(지역/테마)
- USM-DISC-002: 내 주변(현재 위치) 기준으로 피처드 코스 우선 노출
- USM-DISC-003: 대표/여행형 GPX 코스도 동일 상세 흐름으로 진입

Next

- USM-DISC-101: 필터 추천(시간/난이도/태그/화장실 밀도)
- USM-DISC-201: 자연어로 코스 추천 받기(에이전트)

### 3.2 코스 상세 확인

Must(MVP)

- USM-DET-001: 코스 지도(Polyline) + 거리/예상시간 요약 보기
- USM-DET-002: 경로 주변 화장실 요약(toiletCount 등) 보기
- USM-DET-003: 태그/특이사항(경고) 보기

Next

- USM-DET-101: 카페/급수/편의점 등 카테고리 확장
- USM-DET-102: 커뮤니티 검증(최근 탑승/제보) 표시

### 3.3 라이딩(코스 따라가기)

## 4. MVP 컷(정의)

MVP는 아래가 "끝까지" 연결되어야 한다:

- 피처드 코스 목록 -> 코스 상세(요약/경고/화장실) -> 코스 따라가기(레벨2)
- 라이딩 기록 -> 코스 생성 -> 공유 링크 -> 공유 코스 열람

## 5. vNext 제안(개발 순서)

1) Trip Collection + Highlights: 여행 도메인과 코스 신뢰도를 먼저 강화

2) Safety Live Share + Route Package: 실제 주행 중 안심과 사용성 강화

3) Collection complete / challenge: 리텐션 확보

4) 필터 기반 추천(LLM 없이): 데이터/지표가 쌓여야 AI도 의미가 생김

5) 자연어 추천 에이전트(LLM): "자연어 -> 추천 조건(JSON)" 변환 + 결과 설명

6) 턴바이턴: 비용/약관/데이터 품질을 검토 후 결정(외부 앱 딥링크 vs 내장)
