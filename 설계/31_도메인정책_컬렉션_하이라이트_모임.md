## 1. 컬렉션 정책

- 컬렉션 생성은 로그인 사용자만 가능하다.
- 컬렉션 기본 공개 범위는 `private`다.
- `mine=true` 목록 조회는 인증이 필요하다.
- 공개 목록은 현재 구현 기준 `public` 컬렉션만 노출한다.
- 상세 조회는 소유자가 아니면서 `private`인 경우 금지한다.
- 컬렉션 수정과 코스 추가는 소유자만 가능하다.
- 같은 컬렉션에 같은 코스를 중복 추가할 수 없다.
- `positionIndex`가 없거나 음수면 맨 뒤 순번으로 자동 배치한다.

## 2. 하이라이트 정책

- 하이라이트 조회는 공개 읽기를 허용하되, 작성자 본인의 private 하이라이트는 추가로 함께 본다.
- 하이라이트 작성은 로그인 사용자만 가능하다.
- 하이라이트는 대상 코스가 존재해야 생성할 수 있다.
- 하이라이트는 `type`, 좌표(`lat/lon`), `title 또는 note` 중 최소 하나를 요구한다.
- 하이라이트 기본 공개 범위는 `public`이다.
- private 하이라이트는 작성자 본인에게만 보인다.

## 3. 모임 정책

- 모임은 코스를 기반으로 생성되는 확장 기능이다.
- 모임 생성은 로그인 사용자만 가능하다.
- 모임 생성 시 host는 자동으로 참가자 목록에 들어간다.
- 새 모임의 초기 상태는 `OPEN`이다.
- 모임 참가/탈퇴는 인증 사용자만 가능하다.
- 상태가 `OPEN`이 아니면 참가할 수 없다.
- 같은 사용자의 중복 참가 요청은 idempotent 성공으로 처리한다.
- 정원이 있으면 현재 참가자 수가 정원 이상일 때 참가를 거절한다.
- 모임장은 탈퇴할 수 없다.
- `status` 필터는 기본값이 `OPEN`이며, `all`일 때만 전체를 조회한다.

### 3-1. 모임 정책 코드

| 정책코드 | 항목 | 정책 정의 | 현재 구현 기준 |
|---|---|---|---|
| MTP-P-001 | 생성 인증 | 모임 생성은 인증된 사용자만 가능하다. | 구현됨 |
| MTP-P-002 | 초기 상태 | 새 모임의 상태는 `OPEN`이다. | 구현됨 |
| MTP-P-003 | host 자동 참가 | 모임 생성 시 host는 자동 참가자로 저장된다. | 구현됨 |
| MTP-P-004 | 참가 가능 상태 | `OPEN` 상태가 아니면 참가할 수 없다. | 구현됨 |
| MTP-P-005 | 중복 참가 | 동일 사용자의 중복 참가 요청은 idempotent 성공으로 처리한다. | 구현됨 |
| MTP-P-006 | 정원 제한 | 정원이 가득 찬 모임은 참가를 거절한다. | 구현됨 |
| MTP-P-007 | 탈퇴 제한 | 모임장은 leave할 수 없다. | 구현됨 |
| MTP-P-008 | 상태 필터 기본값 | 목록 조회 상태 필터 기본값은 `OPEN`이며 `all`이면 전체 조회다. | 구현됨 |

## 4. 입력 검증 정책

- 컬렉션 제목은 비어 있을 수 없고 120자를 넘길 수 없다.
- 모임 시작 시간은 현재 시각보다 지나치게 과거일 수 없고 180일을 넘길 수 없다.
- 모임 좌표는 `meetingPointLat/lon`을 쌍으로 전달해야 한다.
- 모임 정원은 1 이상 100 이하만 허용한다.

| 정책코드 | 항목 | 정책 정의 | 현재 구현 기준 |
|---|---|---|---|

| VAL-P-002 | 모임 시작 시간 | `startAt`은 현재 시각 이후이며 180일 이내여야 한다. | 구현됨 |
