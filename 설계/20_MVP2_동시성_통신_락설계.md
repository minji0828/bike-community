## 1. 동시성 요구(정의)

MVP2에서 동시성이 중요한 이유:

- "정원 8명" 모임에 20명이 동시에 join하면,
  - 정원이 초과되면 안 되고,
  - 같은 기기가 중복 참여 처리되면 안 되고,
  - 네트워크 재시도/중복 탭에도 결과는 멱등해야 한다.

---

## 2. 권장 트랜잭션 패턴: Row Lock(SELECT FOR UPDATE)

### 2.1 왜 이게 기본값인가

- 분산락/Redis 없이도 Postgres row-level lock으로 안전하게 직렬화 가능
- 설계/구현이 직관적이라 오류 확률이 낮음(1인 MVP2에 적합)

근거:

- Postgres row-level locking: https://www.postgresql.org/docs/current/explicit-locking.html#LOCKING-ROWS

### 2.2 Join 처리(의사코드)

트랜잭션 내부에서 수행:

```text
1) SELECT * FROM course_meetups WHERE id = :meetupId FOR UPDATE;
2) if status != 'open' -> 409/410
3) if exists(select 1 from participants where meetup_id=:id and device_uuid=:device) -> 200 (already joined)
4) if participant_count >= max_participants -> 409 (full)
5) INSERT participants(meetup_id, device_uuid, role, joined_at) ...
6) UPDATE course_meetups SET participant_count = participant_count + 1 WHERE id=:id;
7) COMMIT
```

주의:

- (중요) "이미 참가자" 체크는 락 안에서 해야 한다.
- participants PK(meetup_id, device_uuid)로 중복 삽입은 최종 방어한다.

### 2.3 Leave 처리(의사코드)

```text
1) SELECT meetup FOR UPDATE
2) if not exists participant -> 200 (already left)
3) UPDATE participant set left_at=now
4) UPDATE meetup set participant_count = participant_count - 1
```

## 3. 대안 패턴: 원자 UPDATE(카운터) + Unique 제약

### 3.1 언제 쓰나

- 조회수(viewCount) 같은 "정원 제한이 없는" 카운터는 원자 UPDATE가 더 단순하고 효율적이다.

예시:

- `UPDATE courses SET view_count = view_count + 1 WHERE id = :courseId;`

정원(join) 같은 케이스에는 단순 원자 UPDATE만으로는 "중복 join"에서 카운트가 틀어질 수 있어, 주의가 필요하다.

---

## 4. 멱등성(Idempotency)

### 4.1 최소 멱등성(권장)

- join 요청은 "이미 참가"면 200으로 동일 결과를 반환한다.
- DB 제약(PK/unique)로 중복 참가를 막는다.

### 4.2 Idempotency-Key(선택)

- 네트워크 재시도/중복 탭이 잦다면 `Idempotency-Key`를 저장해 정확히 동일 응답을 재사용한다.
- MVP2에서는 선택(복잡도 증가).

---

## 5. 통신 방식(실시간) 선택

### 5.1 MVP2 기본값

- REST + 폴링(5~10초)
  - 구현/운영이 단순
  - 참가자 수/상태는 "조회"로 충분

### 5.2 SSE/WebSocket 도입 기준

- SSE: "상태 변화 알림" 정도만 필요할 때
- WebSocket(STOMP): 채팅/참가자 목록 실시간 갱신이 필요할 때

근거:

- Spring SSE: https://docs.spring.io/spring-framework/reference/web/webmvc/mvc-ann-async.html#mvc-ann-async-sse
- Spring STOMP over WebSocket: https://docs.spring.io/spring-framework/reference/web/websocket/stomp.html
