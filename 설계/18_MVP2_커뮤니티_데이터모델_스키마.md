## 0. 공통 규칙

- 좌표계: SRID 4326
- 좌표 순서: (lon, lat)
- 익명 식별: `device_uuid`(스푸핑 가능, 강한 신뢰 X)

---

## 1. 신규 테이블(제안)

### 1.1 course_meetups

역할: 코스에 딸린 "모임(같이 타기)" 모집 단위.

- id (PK)
- course_id (FK -> courses.id)
- host_device_uuid (not null)
- title (not null)
- start_at (timestamp, not null)
- meeting_point geometry(Point,4326) (nullable)
- max_participants (int, not null)
- participant_count (int, not null, default 0)
- status (enum string: open/closed/cancelled)
- join_code (unique, not null)
- created_at, updated_at

제약/정책:

- max_participants >= 2
- participant_count는 트랜잭션으로만 변경(정원 동시성)

인덱스(권장):

- btree(course_id, start_at)
- btree(join_code) unique
- btree(status, start_at)

### 1.2 course_meetup_participants

역할: 모임 참가자.

- meetup_id (FK -> course_meetups.id)
- device_uuid (not null)
- nickname (nullable)
- role (enum string: host/member)
- joined_at
- left_at (nullable)
- primary key(meetup_id, device_uuid)

인덱스(권장):

- btree(meetup_id)
- btree(device_uuid)

### 1.3 course_comments (MVP2 Next 후보)

역할: 코스에 달리는 댓글/후기.

- id (PK)
- course_id (FK)
- device_uuid (not null)
- body (text, not null)
- status (enum: visible/hidden/deleted)
- created_at
- updated_at

인덱스(권장):

- btree(course_id, created_at desc)
- btree(status)

### 1.4 reports (MVP2 선택)

역할: 모임/댓글/코스 신고.

- id (PK)
- reporter_device_uuid (not null)
- target_type (enum: course/meetup/comment)
- target_id (not null)
- reason (enum string)
- note (nullable)
- status (enum: open/triaged/resolved)
- created_at

인덱스(권장):

- btree(target_type, target_id)
- btree(status, created_at desc)

## 2. 동시성/정합성 메모

### 2.1 정원 모집(join) 정합성

원칙:

- 참가자 중복은 PK(meetup_id, device_uuid)로 차단
- 정원 제한은 "meetup row lock" 기반 트랜잭션으로 처리

권장 흐름(요약):

1) meetup row를 `SELECT ... FOR UPDATE`로 잠금
2) 이미 참가자인지 확인
3) participant_count < max_participants 확인
4) participants insert
5) participant_count + 1 update

자세한 내용은 `설계/20_MVP2_동시성_통신_락설계.md`를 따른다.
