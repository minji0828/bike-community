## 0. 이 문서의 역할

- 이 문서는 제품의 1차 클라이언트인 모바일 앱 구현 기준 문서다.
- 현재 저장소에 모바일 소스는 없지만, 모바일 재착수 시 이 문서를 기준으로 바로 프로젝트를 열 수 있게 한다.
- 현재 저장소의 웹 구현 규칙은 `설계/29_프론트엔드_개발_가이드.md`를 따르며, 이 문서는 제품 기준 모바일 설계를 담당한다.

## 1. 현재 제품 타깃

- 제품의 1차 클라이언트는 모바일 앱이다.
- 기준 스택은 React Native + Expo 계열로 본다.
- 목표 소스 위치는 `frontend/mobile`이다.
- 현재 저장소의 `frontend/` 웹 앱은 공유 열람, 운영 검증, 공개 진입용 보조 구현체로 취급한다.

## 2. 모바일 MVP1 핵심 화면

| Route Group | 화면 | 목적 |
|---|---|---|
| `(auth)` | 로그인 | 카카오 로그인 시작/종료 |
| `(tabs)/home` | 홈 | featured 코스, 주변 화장실 |
| `(tabs)/ride` | 라이딩 기록 | 기록 시작/일시정지/종료/저장 |
| `(tabs)/profile` | 프로필 | 로그인 상태, 내 활동 |
| `course/[id]` | 코스 상세 | 코스 메타, 경고, CTA |
| `course/[id]/guide` | 따라가기 | L2 진행률, 이탈 안내 |
| `share/[shareId]` | 공유 코스 열람 | 외부 링크 진입 |
| `ride/result/[ridingId]` | 저장 완료 | 코스 생성 유도 |

## 3. 상태관리 원칙

- 인증 상태만 전역 컨텍스트로 공유한다.
- 화면 로딩, 선택, 폼, GPS 진행 상태는 기능 단위 로컬 상태를 기본으로 한다.
- 별도 전역 상태관리 라이브러리 도입은 새 ADR 없이 하지 않는다.

## 4. 인증 규칙

- 카카오 로그인은 Authorization Code + PKCE를 기본으로 한다.
- 모바일 앱은 public client이므로 client secret을 앱에 넣지 않는다.
- 백엔드 `/api/v1/auth/kakao`가 code 교환과 `id_token` 검증을 수행한다.
- 모바일은 백엔드 service JWT만 저장하고, 카카오 provider token을 일반 API 토큰으로 사용하지 않는다.
- 저장소는 secure storage를 사용한다. 평문 AsyncStorage 저장은 금지한다.

## 5. 지도/GPS 규칙

- 라이딩 기록과 따라가기는 모바일 위치 권한을 필요로 한다.
- MVP1은 foreground 위치 권한만 요구한다.
- 기본 샘플링은 `timeInterval=2000ms`, `distanceInterval=5m`를 사용한다.
- path는 업로드 전 최대 250 포인트로 다운샘플링한다.
- 오프루트는 코스 선과의 거리 기준으로 판정한다.
- 권한 거부 시 홈/코스 상세는 계속 사용 가능해야 하고, 기록/따라가기는 제한 모드로 전환한다.

## 6. API 연동 규칙

- 모든 API는 `docs/04_API_상태계약.md`의 `code/message/data` envelope를 따른다.
- 모바일 내부 좌표는 `{ lat, lng }`, API 경계는 `{ lat, lon }`로 변환한다.
- 라이딩 저장, 코스 생성, 공유 발급은 보호 API다.
- 공유 코스 조회는 로그인 없이 가능해야 한다.

## 7. 오프라인/복구 원칙

- 라이딩 기록 중 네트워크가 끊겨도 path 메모리는 유지한다.
- 저장 요청 실패 시 사용자가 재시도할 수 있어야 한다.
- MVP1은 전체 오프라인 동기화 큐까지 요구하지 않지만, 저장 실패 재시도 진입점은 반드시 둔다.

## 8. UI 원칙

- 홈은 검색보다 "즉시 탈 코스" 진입을 우선한다.
- 코스 상세의 1차 CTA는 `따라가기`다.
- 라이딩 기록 화면은 숫자 HUD가 우선이고, 복잡한 메타는 후순위다.
- 디자인 토큰은 `설계/33_디자인토큰_가이드.md`를 따른다.

## 9. 구현 시작 체크리스트

- `frontend/mobile` 초기화
- Expo Router 또는 동등 라우터 결정
- secure storage 연동
- Kakao 로그인 시작/콜백/완료 플로우 연결
- 지도 SDK 선택과 L2 가이드 화면 초기 뼈대 구성
- 라이딩 기록 샘플링/다운샘플링 유틸 구현

## 10. 현재 저장소와의 관계

- 현재 저장소 웹 앱에 있는 기능은 모바일 MVP를 검증하기 위한 보조 자산으로 활용한다.
- 모바일과 웹이 같은 API 계약을 사용해야 하므로, API나 상태값을 바꾸면 이 문서와 웹 가이드를 같이 수정한다.
