## 1. 현재 사용 기술(서버)

### 1.1 Spring Boot 3.5.x

- 범위: 서버 프레임워크(웹/DI/설정/런타임)
- 사용 근거:
  - REST API 개발 속도와 표준 패턴(Controller/Service/Repository)을 빠르게 확보
  - springdoc/validation/security 등 필수 컴포넌트와 결합이 쉬움
- 대안:
  - Ktor/Quarkus/Micronaut: 더 가볍거나 빠를 수 있으나 팀/생태계/학습 비용 고려
- 리스크/주의:
  - 설정이 많아질수록 프로파일/환경변수 관리가 중요

### 1.2 Gradle Wrapper

- 범위: 빌드/테스트/실행 표준화
- 사용 근거:
  - 팀/환경 차이 없이 동일 Gradle 버전으로 재현 가능

### 1.3 Spring Web (spring-boot-starter-web)

- 범위: REST API
- 사용 근거:
  - 모바일 앱(RN)과의 HTTP 계약을 빠르게 구현 가능

### 1.4 Spring Data JPA + Hibernate ORM

- 범위: RDB 매핑/트랜잭션
- 사용 근거:
  - 코스/POI/라이딩 같은 CRUD 중심 도메인에서 개발 속도 우수
  - 서비스 레이어 트랜잭션 패턴이 단순
- 대안:
  - MyBatis/jOOQ: 복잡 쿼리/성능 튜닝이 쉬우나 초기 개발 속도/보일러플레이트 증가
- 리스크/주의:
  - N+1 방지(필요 시 fetch 전략/쿼리 최적화)

### 1.5 PostgreSQL + PostGIS

- 범위: 기본 저장소 + 공간 인덱스/공간 질의
- 사용 근거:
  - POI nearby / along-route / 코스 공간 검색은 PostGIS의 인덱스 기반 질의가 표준
  - geometry(LineString/Point) 저장과 `ST_DWithin` 등이 직접 지원됨
- 대안:
  - MongoDB(geoJSON): 단순 주변 검색엔 가능하지만, 고급 공간연산/최적화는 PostGIS가 유리
  - 전용 GIS 엔진: 1인 MVP에 과함
- 리스크/주의:
  - 좌표 순서: (lon,lat) 강제(바다에 찍히는 오류 방지)
  - SRID/geometry vs geography 혼용 시 성능/정확도 주의

### 1.6 Hibernate Spatial

- 범위: JPA에서 PostGIS geometry 타입 지원
- 사용 근거:
  - 표준 JPA만으로는 Point/LineString을 안전하게 매핑하기 어려움

### 1.7 JTS (jts-core)

- 범위: 서버 애플리케이션에서 지오메트리 생성/좌표 처리
- 사용 근거:
  - LineString/Point 생성 및 좌표 변환을 표준 라이브러리로 처리
  - 코스 path를 앱에서 구성해 DB에 저장할 때 필수
- 리스크/주의:
  - Coordinate(x,y) = (lon,lat) 규칙 고정

### 1.8 Spring Security

- 범위: 인증/인가 기반
- 사용 근거:
  - vNext에서 토큰 기반 인증으로 확장할 때 표준 선택
  - 현재는 개발 편의로 느슨하게 두더라도, 기반을 미리 깔아둠

### 1.9 Validation (spring-boot-starter-validation)

- 범위: 요청 DTO 검증
- 사용 근거:
  - 좌표/UUID/필수 필드 누락 같은 입력 오류를 컨트롤러에서 빠르게 차단

### 1.10 springdoc-openapi (Swagger UI)

- 범위: API 문서 자동화/테스트 UI
- 사용 근거:
  - RN 개발/디버깅을 위해 "실제 서버 계약"을 빠르게 확인 가능

### 1.11 로깅(SLF4J)

- 범위: 운영/디버깅 로그
- 사용 근거:
  - 비동기 작업/동기화/시드 로딩 같은 플로우 추적에 필수

## 2. 파일 포맷/프로토콜

### 2.1 GPX 1.1

- 범위: 코스 경로의 교환 포맷(업로드/다운로드)
- 사용 근거:
  - 자전거 커뮤니티/서비스(Strava 등)에서 널리 쓰이는 사실상 표준 포맷
  - 앱에서 코스를 공유/백업/재활용하기에 적합
- 현재 구현 정책:
  - DB에는 (a) `path`(LineString, 공간 질의/계산용) + (b) `gpx_data`(원본 XML, 다운로드/재공유용)를 함께 보관 가능
  - GPX 파싱은 JDK XML 파서(DOM)로 `trkpt(lat,lon)`만 최소 파싱한다(추가 확장 태그는 무시)
- 대안:
  - GPX 전용 파서 라이브러리: 더 풍부한 지원 가능(트랙/라우트/웨이포인트, 확장 태그)하지만 MVP에 과할 수 있음
- 리스크/주의:
  - XML 보안(XXE 등) 방지 설정 필요(secure processing + doctype 금지)

## 2.2 실시간 통신(SSE / WebSocket) (vNext)

적용 후보: 코스 모임(모집) 참여자 변화 알림, 모임 채팅.

- 옵션 A: REST + 폴링
  - 장점: 구현 가장 단순
  - 단점: 실시간 체감 약함/트래픽 증가
- 옵션 B: SSE(Spring MVC)
  - 장점: 서버->클라이언트 단방향 알림에 적합
  - 단점: 모바일 환경에서 연결 안정성/재연결 처리 필요
- 옵션 C: WebSocket + STOMP(Spring)
  - 장점: pub-sub/채팅 등 양방향에 적합
  - 단점: 스케일링/브로커/인증 고려가 커짐

근거 링크:

- Spring STOMP over WebSocket: https://docs.spring.io/spring-framework/reference/web/websocket/stomp.html
- Spring SSE: https://docs.spring.io/spring-framework/reference/web/webmvc/mvc-ann-async.html#mvc-ann-async-sse

## 2.3 동시성 제어(Postgres Lock / JPA @Version) (vNext)

적용 후보: 모임 정원 참가(join), 카운터(viewCount/followCount) 증가.

- 옵션 A: Postgres row-level lock / 원자 UPDATE
  - 장점: 분산락 없이도 동시성 안전하게 처리 가능
  - 단점: 트랜잭션/락 범위 설계 필요
- 옵션 B: JPA Optimistic Locking(@Version)
  - 장점: 충돌 감지/재시도 패턴이 명확
  - 단점: 충돌 빈도가 높으면 재시도 비용 증가

근거 링크:

- Postgres row-level locking: https://www.postgresql.org/docs/current/explicit-locking.html#LOCKING-ROWS
- Spring Data JPA locking: https://docs.spring.io/spring-data/jpa/reference/jpa/locking.html

## 2.4 인증(OAuth2/OIDC) - Kakao (MVP2)

- 범위: 소셜 로그인(유저 편의) + write API 보호(댓글)
- 사용 근거:
  - 비로그인(deviceUuid) 기반은 스푸핑/스팸에 취약
  - 댓글/신고/삭제 같은 행위는 최소한의 계정 식별이 있어야 운영이 가능
- 권장 플로우:
  - Authorization Code + PKCE(RN)
  - 서버에서 토큰 교환 + id_token 검증 + 서비스 JWT 발급
- 리스크/주의:
  - 앱에 client_secret 저장 금지
  - id_token 검증(iss/aud/signature/exp) 필수

근거 링크:

- Kakao Login REST API: https://developers.kakao.com/docs/latest/en/kakaologin/rest-api
- Kakao OIDC discovery: https://kauth.kakao.com/.well-known/openid-configuration
- Spring OAuth2 Login: https://docs.spring.io/spring-security/reference/servlet/oauth2/login/index.html
- Spring OIDC: https://docs.spring.io/spring-security/reference/servlet/oauth2/login/core.html#oauth2login-oidc
- Spring Resource Server JWT: https://docs.spring.io/spring-security/reference/servlet/oauth2/resource-server/jwt.html
