## 1. 토큰 원칙

- 토큰은 의미 기반 이름을 사용한다. 예: `primary`, `muted`, `destructive`.
- 화면은 raw 색상값이 아니라 의미 토큰을 참조한다.
- 웹과 모바일은 표현 수단이 달라도 같은 토큰 이름을 공유한다.
- 다크모드는 별도 색상표가 아니라 같은 의미 토큰의 다른 값이다.

---

## 2. 현재 기준 색상 토큰

| 토큰 | 의미 | 현재 웹 값 |
|---|---|---|
| `background` | 앱 기본 배경 | `oklch(0.98 0.01 160)` |
| `foreground` | 기본 본문 텍스트 | `oklch(0.25 0.02 200)` |
| `card` | 카드 표면 | `oklch(1 0 0)` |
| `primary` | 핵심 CTA, 강조 | `oklch(0.65 0.18 160)` |
| `secondary` | 보조 강조 | `oklch(0.95 0.03 85)` |
| `muted` | 중립 배경/보조 정보 | `oklch(0.95 0.01 160)` |
| `accent` | 주목 포인트 | `oklch(0.85 0.15 85)` |
| `destructive` | 위험/삭제 | `oklch(0.577 0.245 27.325)` |
| `border` | 경계선 | `oklch(0.9 0.02 160)` |
| `ring` | 포커스 표시 | `oklch(0.65 0.18 160)` |

메모:

- 현재 웹 기본 팔레트는 녹색 계열 primary와 모래색 secondary를 사용한다.
- 모바일 앱도 같은 의미 팔레트를 유지하되, 네이티브 환경의 대비 기준에 맞게 값 보정은 가능하다.

## 3. 차트/지도 보조 토큰

| 토큰 | 의미 |
|---|---|
| `chart-1` | 핵심 메트릭 강조 |
| `chart-2` | 보조 메트릭 |
| `chart-3` | 상태 변화 보조 |
| `chart-4` | 주의/경고 보조 |
| `chart-5` | 확장 시리즈 |

지도/가이드 권장 의미 매핑:

- 현재 위치 점: `primary`
- 경로 선: `chart-1`
- 이탈 경고: `destructive`
- 편의시설 핀: `accent`

---

## 4. 타이포그래피 토큰

| 토큰 | 역할 | 현재 기준 |
|---|---|---|
| `font-sans` | 기본 UI 텍스트 | `Geist` 계열 |
| `font-mono` | 좌표, 코드, 메트릭 | `Geist Mono` 계열 |

타이포 원칙:

- 화면 제목은 짧고 강하게, 본문은 밀도보다 가독성을 우선한다.
- 숫자와 메트릭은 monospaced 폰트를 우선 검토한다.
- 모바일은 시스템 폰트 폴백을 두되 토큰 이름은 유지한다.

---

## 5. 라운드/간격 토큰

| 토큰 | 역할 | 현재 기준 |
|---|---|---|
| `radius` | 기본 컴포넌트 모서리 | `1rem` |
| `radius-sm` | 작은 chip/input | `radius - 4px` |
| `radius-md` | 기본 버튼/input | `radius - 2px` |
| `radius-lg` | 카드/패널 | `radius` |
| `radius-xl` | hero panel / modal | `radius + 4px` |

간격 원칙:

- 화면 섹션은 24~32px 단위, 카드 내부는 12~16px 단위를 기본으로 한다.
- 모바일는 세로 리듬을 유지하고, 웹은 가로 여백을 더 준다.

---

## 6. 상태 토큰 매핑

| 상태 | 사용 토큰 |
|---|---|
| 기본 CTA | `primary` |
| 보조 CTA | `secondary` |
| 비활성 | `muted`, `muted-foreground` |
| 오류 | `destructive` |
| 성공 | `primary` 또는 `chart-2` |
| 경고 | `accent` |

가이드 화면 상태 매핑:

- `playing`: `primary`
- `paused`: `secondary`
- `off_route`: `destructive`
- `finished`: `chart-2`

---

## 7. 구현 규칙

- 웹은 `frontend/app/globals.css`를 현재 토큰 SSOT로 사용한다.
- 모바일 구현 시 동일 의미의 토큰 객체를 `theme/tokens.ts` 같은 파일로 분리해 유지한다.
- 새 색상값을 넣기 전에 기존 의미 토큰으로 해결 가능한지 먼저 검토한다.
- 컴포넌트 코드에서 hex 값을 직접 쓰는 경우는 일회성 프로토타입을 제외하고 금지한다.
