# 프론트엔드 ADR 인덱스

- 범위: 모바일 앱과 웹 클라이언트의 기술 선택, 라우팅, 인증 플로우, 상태관리, 지도, 실시간 통신
- 번호 정책: 프론트엔드 폴더 안에서 독립적으로 증가
- 템플릿: `설계/adr/frontend/_template.md`

## ADR 목록

| ID | 상태 | 제목 | 한 줄 요약 |
|---|---|---|---|
| 0001 | Superseded | 현재 저장소의 프론트엔드 구현체는 Next.js 웹 앱 | 당시 저장소 현실을 설명하는 기록이며 제품 1차 클라이언트 판단은 더 이상 담당하지 않는다 |
| 0002 | Accepted | Kakao 로그인 오케스트레이션은 Next route + backend JWT로 구성 | 프론트는 PKCE를 시작하고 토큰 검증과 발급은 백엔드가 담당한다 |
| 0003 | Accepted | API 연동과 클라이언트 상태는 얇은 래퍼 + 로컬 상태를 기본으로 한다 | 공통 fetch 래퍼와 AuthProvider만 공유 상태로 두고 별도 전역 상태관리 라이브러리는 도입하지 않는다 |
| 0004 | Accepted | 지도는 Kakao Maps JS, 실시간 채팅은 경량 STOMP 클라이언트 | 지도와 실시간은 브라우저 전용 클라이언트 컴포넌트로 한정해 유지한다 |
| 0005 | Accepted | 코스 핵심경로는 공유 링크와 브라우저 GPS 가이드를 우선 | 상세 -> 공유 -> 가이드 흐름을 웹 기준으로 먼저 완성한다 |
| 0006 | Accepted | 제품의 1차 클라이언트는 모바일이며 현재 저장소 웹 앱은 보조 구현체다 | 모바일 제품 방향과 현재 저장소 웹 구현 현실을 분리해 기록한다 |
