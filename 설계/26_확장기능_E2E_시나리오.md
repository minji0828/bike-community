## 1. 목적

확장 기능을 추가할 때 "화면은 있는데 흐름이 끊기는" 상태를 방지하기 위해,
기능별 E2E 완료 기준을 미리 정의한다.

## 2. E2E 시나리오 묶음

### E2E-A. Trip Collection 생성 → 공유 → 코스 소비
1. 운영/사용자가 Trip Collection을 생성한다.
2. 컬렉션에 코스 2개 이상을 순서대로 추가한다.
3. 컬렉션 상세에서 총 거리/기간/대표 포인트를 본다.
4. 공유 링크를 발급한다.
5. 다른 사용자가 링크로 컬렉션을 연다.
6. 컬렉션 내부 코스 중 하나를 따라가기로 진입한다.

완료 기준:
- 컬렉션 상세 진입 성공
- 링크 공유 성공
- 내부 코스 이동 성공
- 비공개 컬렉션 접근 차단

### E2E-B. Highlight 작성 → 노출 → 신고
1. 로그인 사용자가 코스 위 특정 지점에 Highlight를 남긴다.
2. 다른 사용자가 코스 상세에서 Highlight를 본다.
3. 부적절한 Highlight를 신고한다.
4. 운영이 숨김 처리한다.
5. 일반 상세에서는 더 이상 노출되지 않는다.

완료 기준:
- 좌표 기반 Highlight 작성/조회 성공
- 신고 → 운영 숨김 → 일반 비노출 흐름 성공

### E2E-C. Live Share 시작 → 수신자 확인 → 만료
1. 라이딩 시작 후 Live Share를 생성한다.
2. 공유 링크를 수신자가 연다.
3. 수신자는 현재 위치/마지막 갱신/예상 도착 정보를 확인한다.
4. 라이딩 종료 또는 만료 시 링크가 비활성화된다.

완료 기준:
- 비로그인 뷰어도 제한된 정보만 확인 가능
- 만료 후 링크 접근 차단
- 소유자 종료 시 즉시 비활성화

### E2E-D. Route Package 저장 → 오프라인 진입
1. 사용자가 코스 패키지를 저장한다.
2. 네트워크 불안 상태에서 코스 상세/가이드를 연다.
3. geometry, 핵심 POI, warning, meetup basic info가 유지된다.

완료 기준:
- 패키지 저장 성공
- 오프라인/저연결 상태에서도 최소 안내 유지
- stale data 고지 노출

### E2E-E. Collection Complete / Challenge 달성
1. 사용자가 컬렉션 내 코스를 1개씩 완주한다.
2. 진행률이 누적된다.
3. 마지막 코스 완주 시 배지/완료 상태가 반영된다.
4. 모임 완주 기록이 있으면 컬렉션 완료와 연결된다.

완료 기준:
- 중복 완주 집계 정책 일관
- 완료 이벤트 1회만 발생
- 뱃지/피드/상세 상태가 일관

## 3. 공통 테스트 포인트

### 인증
- 비로그인 사용자의 허용 범위/차단 범위가 명확해야 한다.
- 공개/링크공유/비공개 범위가 문서 정책과 일치해야 한다.

### 권한
- Collection 공동 편집자/소유자/일반 사용자 권한 차이 검증
- Highlight 작성자/신고자/운영자 권한 차이 검증

### 실시간
- Live Share와 모임 채팅이 동시에 켜져도 소켓/세션이 충돌하지 않아야 한다.
- reconnect/reload 후 상태 복구가 가능해야 한다.

### 데이터 정합성
- 코스 삭제/비노출 시 Collection, Highlight, Share 링크의 정합성이 유지돼야 한다.
- 모임 종료/삭제 시 Live Share/채팅/집결 정보도 일관되게 정리돼야 한다.

## 4. 릴리즈 게이트

확장 기능은 아래 4개가 모두 만족할 때만 배포 후보로 본다.
- API/정책 문서 갱신 완료
- happy path E2E 1개 이상 자동/반자동 검증
- 실패 path(권한/만료/네트워크) 검증
- 운영 대응(숨김/만료/비노출) 검증
