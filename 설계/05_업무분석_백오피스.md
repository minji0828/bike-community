# 업무분석 & 백오피스(관리) 구성

- 버전: v0.3
- 작성일: 2026-02-25
- 상태: 초안(코스 메타데이터/큐레이션 반영)

## 1. 관리자 역할(Role)

- Admin: POI 동기화/상태 확인, 사용자/데이터 관리
- Curator: 기본 제공 코스(큐레이션) 등록/편집, 태그/특이사항 관리
- Operator(선택): 조회 중심, 제한된 조작(동기화 상태 확인 등)

## 2. 핵심 백오피스 기능(최소)

### BO-001 POI 동기화 실행/상태

- 버튼: 전체 새로고침, 증분 동기화
- 상태: 마지막 성공 시간, 마지막 실패 원인, 처리 건수

### BO-002 사용자 위치 데이터 관리

- 사용자별 위치 이력 조회
- 기간별 삭제(정책에 따른 보관)

### BO-003 라이딩 데이터 조회

- 라이딩 목록/상세
- 이상치(좌표 오류, 거리 0 등) 탐지(선택)

### BO-004 코스 관리(추후)

MVP에서도 "기본 제공 코스" 운영을 위해 최소한의 관리 기능이 필요하다.

- 코스 생성(큐레이션): GPX/좌표 업로드 또는 지도에서 경로 입력(구현 방식은 후순위)
- 코스 편집: 제목/설명/태그/공개범위/대표 이미지(선택)
- 코스 비공개 처리/삭제: 정책에 따른 권한
- 코스 sourceType/verifiedStatus 관리(예: curated)

### BO-005 코스 태그/사전 관리

추천/필터/검색에서 태그 품질이 곧 UX 품질이다.

- 태그 사전 관리: 태그 추가/비활성화, 표시명, 카테고리(예: 분위기/난이도/환경)
- 동의어/유사어 매핑(선택): "강변" ~ "하천" 같은 그룹화

### BO-006 코스 특이사항(경고) / 하이라이트 관리

`설계/02_MVP_요구사항정의서.md`의 경고 구조를 기준으로 운영한다.

- 경고 유형/심각도(type/severity) 사전 관리
- 코스별 경고 추가/수정/만료(validUntil)
- 하이라이트(뷰포인트/정차 추천) 관리(선택)

### BO-007 코스 메타데이터(파생치) 재계산/검증

서버가 자동 산출하는 메타데이터(distance/bbox/amenitiesSummary 등)를 관리자가 재계산할 수 있어야 한다.

- 단건 재계산: 특정 코스 1개
- 배치 재계산: featured/최근 코스 우선
- 마지막 계산 시간/버전/실패 원인 확인

### BO-008 기본 제공(피처드) 코스 운영

- 지역별 기본 코스 세트 구성(예: 서울/한강, 세종, 제주 등)
- 노출 순서(featuredRank) 설정
- 임시 비노출(공사/안전 이슈) 처리

### BO-009 코스 품질/성능 관리(추후)

- 좌표가 비정상적으로 많은 코스 탐지(렌더링/저장 비용)
- 코스 주변 POI 조회가 느린 구간/코스 모니터링

## 4. 운영 시나리오

- 외부 API 장애 -> 증분 실패 -> 재시도 -> 알림(추후) -> 전체 새로고침 고려
- 개인정보 삭제 요청 -> 사용자 위치/라이딩 데이터 범위 확인 -> 삭제 실행 -> 로그 보관

- 기본 코스(큐레이션) 추가:
  - 코스 등록 -> 태그/특이사항 입력 -> 파생치 계산 -> featuredRank 설정 -> 앱 노출 확인
- 도로 공사/안전 이슈:
  - 코스 경고 추가(validUntil 포함) 또는 임시 비노출 -> 재검증 후 복구
