# Plans

앞으로 작업 계획 문서는 이 폴더에 Markdown으로 남깁니다.

## 규칙

- 파일명: `YYYY-MM-DD_<주제>_plan.md`
- 목적:
  - 구현 전에 범위/우선순위/검증 기준을 고정
  - 이후 work-report와 분리해서 계획과 결과를 따로 추적
- 권장 구성:
  1. 목표
  2. 범위
  3. 비범위
  4. 작업 순서
  5. 검증 기준
  6. 리스크

## 참고

- 기존 OMX 내부 계획 문서는 `.omx/plans/`에 남아 있습니다.
- 앞으로 사용자와 합의한 실행 계획의 기준본은 `plans/`에 우선 기록합니다.
