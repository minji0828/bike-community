# MVP2 요구사항 정의서: 카카오 소셜 로그인(OIDC) + 코스 댓글

- 문서 ID: MVP2-REQ
- 버전: v0.1
- 작성일: 2026-03-04
- 상태: 초안(설계 확정용)

목적:

- MVP1이 "코스 콘텐츠"를 만들고 따라가는 것이라면, MVP2는 "코스에 사람이 남긴 흔적"을 만든다.
- 카카오 소셜 로그인만 도입하고, 코스 댓글(후기/제보)로 커뮤니티성을 만든다.
- 모임/채팅은 MVP2에서 제외하고 MVP3/vNext로 미룬다.

관련 문서:

- MVP1 범위: `설계/02_MVP_요구사항정의서.md`

- 기술 근거(실시간/락): `설계/15_기술스택_레지스트리.md`

## 0. MVP2의 한 줄

카카오로 로그인한 사용자가 코스에 댓글을 남기고, 코스 상세가 "살아 있는 페이지"처럼 느껴지게 한다.

---

## 1. 목표(Goals)

- G1 커뮤니티 체감: 코스에 댓글/후기가 쌓여 "혼자" 느낌을 줄인다.
- G2 사용자 편의: OIDC 기반 소셜 로그인으로 작성/신고 같은 행위를 가능하게 한다.
- G3 운영 안전: 스팸/도배를 최소 기능으로 제어한다(레이트리밋/숨김).

---

## 2. 이번 MVP2에서 제외(Non-Goals)

- 팔로우/피드/알고리즘 추천 같은 "소셜 네트워크" 풀세트
- 코스 모임(모집/정원/참여)
- 채팅(WebSocket)
- 실시간 위치 공유
- 실명/프로필/친구 관계(소셜 네트워크)

---

## 3. 핵심 사용자 시나리오

### S1 카카오 로그인 -> 댓글 작성

- 사용자는 카카오로 로그인한다.
- 코스 상세에서 짧은 댓글(후기/제보)을 작성한다.
- 작성 후 코스 상세에서 최신 댓글이 바로 보인다.

### S2 댓글 신고/숨김(운영 최소)

- 사용자는 스팸/유해 댓글을 신고한다.
- 운영은 해당 댓글을 숨김 처리할 수 있다.

---

## 4. 기능 범위 (MVP2 Must)

### A. 인증(카카오 OIDC)

- A1 로그인: RN에서 받은 authorization code(PKCE)를 서버로 전달하고, 서버는 카카오 토큰을 검증한 뒤 서비스용 토큰을 발급한다.
- A2 API 보호: 댓글 작성/삭제/신고 같은 write는 인증 필요.

### B. 코스 댓글

- B1 댓글 작성: 코스 상세에서 댓글을 작성한다.
- B2 댓글 조회: 코스 상세에서 최신 댓글 리스트를 본다.
- B3 댓글 삭제: 작성자는 자신의 댓글을 삭제할 수 있다(soft delete 권장).
- B4 익명 표시: 댓글 작성자는 UI에서 항상 "익명"으로 표시한다(작성자 식별 정보 노출 금지).

### C. 최소 운영/안전

- C1 Rate limit: 댓글 작성/신고를 제한한다.
- C2 신고: 스팸/유해 댓글 신고를 남길 수 있다.

---

## 5. 기능 범위 (MVP2 Next)

- 코스 모임(모집/정원/참여)
- 모임/그룹 임시 채팅(WebSocket)
- "최근 탑승" 같은 소셜 프루프(집계)

---

## 6. MVP2 기술/설계 결정

### 6.1 로그인 방식

- 카카오 OAuth2 + OIDC(Authorization Code + PKCE)
- 서버는 id_token(JWT)을 검증하고(issuer/audience/signature), 서비스 토큰을 발급한다.

상세 설계:

- `설계/21_인증_카카오_OIDC.md`

### 6.2 통신(실시간)

- MVP2는 실시간 통신이 필수는 아니다(댓글은 폴링/재조회로 충분).
- 채팅/모임은 MVP3에서 WebSocket(STOMP)로 도입한다.

---

## 7. 성공 지표(초안)

- M1: 댓글 작성 수(주간)
- M2: 코스 상세 -> 댓글 작성 전환율
- M3: 코스 상세 재방문(7일)

---

## 8. 오픈 이슈(결정 필요)

- O1 댓글은 코스당/일자당 제한을 어떤 수준으로 둘지
- O2 댓글은 "수정"을 허용할지(기본: 불허, 삭제+재작성)
- O3 서비스 토큰은 JWT로 갈지(권장), 세션 쿠키로 갈지
