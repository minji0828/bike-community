# Bike Oasis 설계 문서 (Notion 프로젝트 홈)

## 목적

- `설계/` 폴더의 문서를 Notion에서 **프로젝트 위키**처럼 탐색/공유하기 위한 페이지 트리입니다.
- 원본 단일 진실원천(SSOT)은 레포의 `설계/`이며, 변경은 원본 문서를 먼저 갱신하는 흐름을 유지합니다.

## 한 줄 요약

- 기본 제공(피처드) 코스를 보여주고, 코스 주변의 필수 편의(우선: 화장실)를 함께 제시하며, 코스를 따라갈 수 있는(레벨 2) 자전거 산책 앱

## 페이지 트리(섹션)

- 01_플레이북: 문서 운영/개발 플레이북
- 02_제품: 문제정의/가설, 아이데이션, 사용자 관점(USM/플로우)
- 03_요구사항_정책: MVP 범위, FR/NFR, 정책(규칙)
- 04_도메인_데이터: 도메인 경계, 이벤트 흐름, DB 스키마
- 05_API_인증: API 계약, 인증(OIDC)
- 06_아키텍처_기술: 런타임 플로우, 기술 선택 근거, 기술 레지스트리
- 07_운영_계획: 백오피스/운영, 마일스톤/체크리스트
- 08_MVP2_MVP3: 커뮤니티/동시성/락, 모임/채팅(vNext)
- 09_라이딩: GPS 추적/기록 설계

## 변경 규칙(요약)

- API 변경: `설계/11_API_명세.md` 먼저 갱신
- DB 변경: `설계/10_데이터모델_스키마.md` 먼저 갱신
- 정책 변경: `설계/04_정책정의서.md` 먼저 갱신
- 지오메트리 좌표 순서: DB/JTS/WKT는 (lon, lat) 고정
