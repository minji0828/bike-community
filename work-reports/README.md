# Work Reports (작업 일지/결과 보고)

목적:

- 작업 내용을 일자별로 기록해, "무엇을 했는지/왜 했는지/어떻게 검증했는지"를 빠르게 복기할 수 있게 한다.
- 최종 목표(마일스톤) 대비 현재 진척도를 매 작업마다 남긴다.

작성 규칙:

- 작성 시점: 작업 단위(테스트 통과 + 커밋/푸시) 완료 직후
- 위치: `work-reports/`
- 파일명: `YYYY-MM-DD_HHMM_<short-title>.md` (KST)
- 주의: 시크릿/토큰/개인정보(정확한 위치 포함)는 절대 기록하지 않는다.

---

## 보고서 양식(템플릿)

```md
# 작업 일지 / 결과 보고서

- 작성일시: YYYY-MM-DD HH:mm:ss (KST)
- 작업명: ...
- 브랜치: ...
- 범위(커밋): <old>..<new>

## 1. 이번 작업의 목표

- ...

## 2. 완료된 작업(Deliverables)

- Backend:
- Mobile:
- Docs:

## 3. 변경된 API / 정책(있으면)

- ...

## 4. 검증(Verification)

- 실행한 커맨드:
  - ...
- 결과:
  - ...

## 5. 마일스톤 진척도(설계/14 기준)

- 기준 문서: `설계/14_개발계획_마일스톤_체크리스트.md`
- 산출 방식(권장): Done=1, Partial=0.5, Todo=0
- 이번 보고 시점 진척도: NN%

## 6. 리스크/이슈/메모

- ...

## 7. 다음 작업(Next)

- ...
```

---

## 진척도(%) 산출 가이드(권장)

`설계/14_개발계획_마일스톤_체크리스트.md`의 마일스톤(M0~M7) 상태를 다음처럼 환산해 계산한다.

- Done = 1
- Partial = 0.5
- Todo = 0

예시:

- 완료 합계 / 전체 마일스톤 수 = %

메모:

- 한 번의 작업에서 "체감 진척"이 크게 움직이는 구간이 있으므로, %는 절대값보다 추세(증감)를 보는 용도로 사용한다.
