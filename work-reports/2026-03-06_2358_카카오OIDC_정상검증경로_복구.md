## 1. 이번 작업의 목표

- 카카오 OAuth/OIDC 공식 문서 기준으로 현재 로그인 구현을 재검토한다.
- userinfo fallback 중심 흐름이 아니라, id_token 서명 검증 기반의 정상 로그인 경로로 복구한다.
- 프론트 프로필 화면은 로그인 후 닉네임과 로그아웃 중심으로 단순화한다.

## 2. 완료된 작업(Deliverables)

- Backend:
  - Kakao OIDC discovery/JWKS 확인 후 Spring `JwtDecoder` 의존 대신 커스텀 `KakaoIdTokenVerifier`로 전환
  - `KakaoJwkClient` 추가
    - Kakao JWKS를 직접 가져와 캐시
  - `KakaoIdTokenVerifier`
    - `alg=RS256`, `kid`, 서명, `iss`, `aud`, `exp`, `iat`, `sub` 검증 구현
  - `AuthService`
    - id_token 검증 실패 시 userinfo fallback 로그인 제거
    - 검증된 id_token의 nickname 기반으로 username 반영 유지
- Frontend:
  - 프로필 로그인 카드에서 로그인 후 닉네임 + 로그아웃 중심으로 단순화
  - 로그인 사용자 영역에서 불필요한 보조 CTA 제거
- Tests:
  - `KakaoIdTokenVerifierTest` 추가
  - 기존 `AuthServiceTest`를 fallback 제거 기준으로 갱신

## 3. 변경된 API / 정책(있으면)

- 외부 API 계약 변경 없음
- 내부 인증 정책 변경:

  - 이전의 userinfo fallback 로그인 경로 제거
