## 3. 변경된 API / 정책(있으면)

- 인증 정책:
  - `POST /api/v1/ridings`
  - `POST /api/v1/courses`
  - `POST /api/v1/courses/from-riding`
  - `POST /api/v1/courses/gpx`
  - 위 write API는 인증 필수로 고정
- 소유권 정책:
  - 라이딩/코스 소유권은 요청 바디가 아니라 서비스 JWT `sub` 기준으로 확정

- 프론트 UX 정책:
  - 코스 상세 최상단 행동은 `따라가기`, `공유 링크 만들기`
  - 공유 코스는 읽기 전용 public route(`/share/[shareId]`) 사용
  - 가이드는 브라우저 geolocation과 경로 투영으로 진행률/이탈 판정
