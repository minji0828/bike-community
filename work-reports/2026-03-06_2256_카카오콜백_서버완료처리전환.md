## 1. 이번 작업의 목표

- 카카오 로그인 후 callback 화면에서 `카카오 로그인 정보를 확인하고 있어요...` 상태에 머무는 문제의 근본 원인을 분석하고, JS 하이드레이션 의존을 제거해 해결한다.

## 2. 완료된 작업(Deliverables)

- Root Cause 분석:
  - 기존 `/auth/kakao/callback` 페이지는 클라이언트 컴포넌트 + `useEffect` 로만 후속 처리를 수행
  - 시작 버튼도 한 차례 하이드레이션 문제를 겪었고, 실제 로그에서도 callback GET은 들어오지만 backend auth POST가 보장되지 않는 구간이 존재
  - 즉, authorize는 통과했어도 callback 이후 처리가 브라우저 JS 실행에 묶여 있어 정지처럼 보일 수 있었음
- Mobile/Web:
  - `/auth/kakao/callback` 을 서버 페이지로 전환해 즉시 `/auth/kakao/finalize` 로 redirect 하도록 변경
  - `/auth/kakao/finalize` route 추가
    - 로그인 세션 쿠키(state/nonce/codeVerifier/redirectUri) 검증
    - 서버에서 backend `/api/v1/auth/kakao` 호출
    - 성공 시 access token 쿠키 저장 후 `/profile?login=success` redirect
    - 실패 시 `/profile?loginError=...` redirect
  - `frontend/lib/auth.ts` 에 access token cookie fallback 추가
  - `frontend/app/profile/page.tsx` 에 `loginError` query 표시 추가
- Backend:
  - 변경된 `JwtConfig` 가 반영되도록 백엔드 재기동

## 3. 변경된 API / 정책(있으면)

## 6. 리스크/이슈/메모

- 이제 callback 완료는 브라우저 하이드레이션에 의존하지 않지만, 최종 성공 여부는 실제 Kakao code/id_token 발급이 정상이어야 한다.
- access token 은 현재 localStorage + cookie fallback 병행 구조다. 운영 보안 관점에선 향후 HttpOnly cookie 전환 검토 여지가 있다.
